# LC <787> — <Cheapest Flights Within K Stops>
- **Pattern:** <Graphs — Bellman-Ford bounded to `k+1` rounds (per-round snapshot); the "shortest path with a hop limit" template>
- **Brute force:** <DFS/BFS every path from `src`, tracking cost + stops, prune when stops > k, keep the min cost reaching `dst`. Exponential — re-explores overlapping subpaths.>
- **Optimized:** <"≤ k stops" = "≤ k+1 edges." Bellman-Ford's invariant: after `r` relaxation rounds, `dist[v]` = cheapest cost to `v` using **at most `r` edges**. So seed `dist[src]=0` (rest `int.MaxValue`) and run exactly `k+1` rounds; each round, **clone `dist` into `tmp`**, relax every flight `[u,v,w]` off the FROZEN `dist` into `tmp` (`if dist[u]!=MAX && dist[u]+w < tmp[v]`), then `dist = tmp`. Answer is `dist[dst]` (or -1). O((k+1)·E) time, O(n) space.>
- **Key insight:** <The hop cap makes node-cost dominance unsound — a pricier path with *fewer* stops can be the only one that still reaches `dst` in time, so plain Dijkstra ("first pop of a node is final") and any single `dist[]` cost-pruning over-prune. Bellman-Ford sidesteps this entirely: the **round count IS the hop limit**, so no per-state (node, stops) bookkeeping is needed.>
- **Edge cases I had to handle:** <`src == dst` → 0 (dist[src]=0, untouched). Unreachable within k+1 edges → stays MaxValue → return -1. Guard `dist[u]==int.MaxValue` before relaxing or `MAX + w` overflows. Disconnected nodes stay MAX.>
