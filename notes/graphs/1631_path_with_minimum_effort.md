# LC 1631 — Path With Minimum Effort
- **Pattern:** Graphs — minimax path / modified Dijkstra (min-heap by "effort so far")
- **Brute force:** Binary search on the answer `k` ∈ [0, maxHeight diff]: for each candidate `k`, BFS/DFS from (0,0) using only edges where `|height[u]-height[v]| <= k`; smallest feasible `k` that reaches (n-1,m-1) is the answer. O(n·m·log(maxHeight)). Works, but the Dijkstra variant is cleaner and bounds-independent.
- **Optimized:** Dijkstra where the path cost is the **maximum** edge difference along the path, not the sum. `dist[i,j]` = minimum possible "max edge difference" to reach that cell. Relax a neighbor with `ne = max(e, |height[i][j] - height[ni][nj]|)`; push `(ni, nj, ne)` when `ne < dist[ni,nj]`. Min-heap keyed by effort, lazy deletion via `if (e > dist[i,j]) continue;`. First pop of the destination is finalized ⇒ early return. O(n·m·log(n·m)) time, O(n·m) space.
- **Key insight:** It's a **minimax** path problem — replace Dijkstra's `dist + w` relaxation with `max(dist, w)`. The greedy/heap property still holds because `max` is monotonic, so the first time the destination is popped its effort is optimal. Union-Find (Kruskal: sort edges by |diff|, union until source and sink connect) is the elegant alternative, same complexity.
- **Edge cases I had to handle:** 1×1 grid → effort 0 (no edges); single row / single column handled by the general Dijkstra (no special-casing needed); `dist` initialized to `int.MaxValue` with `dist[0,0] = 0`.
- **Where I got stuck and for how long:** First relaxation used only the single edge difference (`dist[ni,nj] = |diff|`) instead of `max(e, |diff|)`, and the comparison was inverted (`diff > dist` against a `MaxValue` seed ⇒ nothing ever updated). Fixed by carrying running max `e` and comparing `ne < dist[ni,nj]`.
- **Template fragments I reused:** `(int di, int dj)[]` moves array for 4-directional grid neighbors; `PriorityQueue<(...), int>` Dijkstra skeleton with lazy-deletion guard and early-exit-on-destination (same shape as LC 743 Network Delay Time).
