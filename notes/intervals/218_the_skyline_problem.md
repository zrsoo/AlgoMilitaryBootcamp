# LC <218> — <The Skyline Problem>
- **Pattern:** <Sweep line + max-heap with lazy deletion. Output key points where the current max active height changes.>
- **Brute force:** <For each x-coordinate, scan all buildings covering it for the tallest — O(n·coords) / O(n^2).>
- **Optimized:** <Convert the buildings array into an event array. Building start gets +h, building end gets -h at its x coordinate. Sort events: ascending by x coordnate, descending by height (added events come first in case of x coordinate being the same so that we don't remove before adding the same height on the same x coordinate, as that would break the skyline to 0 and back to height in case, e.g. a building ends at 3 with height 5 while another begins at 3 with height 5.). Hold a priority queue of heights and a toRemove dict of heights. Seed pq with (0,0) for skyline end. Iterate events. If height is negative, that means it's ending, so add to toRemove (increment toRemove[-h] so that we keep track of multiple same heights). Otherwise, it means height is beginning, so Enqueue with -h as the factor because we want greatest heights on top. Then it's time to check for endings. While pq still has elements and pq.Peek() has a positive entry in the toRemove dict, Dequeue it and decrement toRemove[pq.Peek()]. After checking for removals, currentMaximum = pq.Peek() (pq.Count never reaches 0 because we seeded it with 0, 0). Keep a previous max. If currentMax != prevMax, emit event, i.e. at x coord x from events, new height is cMax. PrevMax = CMax after.> — O(n log n) time, O(n) space
- **Key insight:** <You never search for the "next max" after a removal — a structure that always surfaces its own max does it for you. SortedDictionary in C# has NO O(log n) max accessor (`.Last()` is LINQ → O(n) enumeration), so use `PriorityQueue` + lazy deletion: never remove a middle element, just discard stale tops when they reach the top.>
- **Edge cases I had to handle:** <Equal-x tie-breaking (start-before-end, taller-start-first, shorter-end-first) — wrong order produces phantom dips/spikes. Multiple buildings same height (need counts, not a set). Descent to ground (heap seeded with 0).>
