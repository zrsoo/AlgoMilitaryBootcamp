# LC <15> — <3Sum>
- **Pattern:** <Two Pointers on a sorted array (kSum family)>
- **Brute force:** <Three nested loops, dedup via sorted-triple `HashSet`> — O(n³) time, O(n) extra for dedup set
- **Optimized:** <Sort, then for each anchor `i` run a two-pointer scan over the suffix to find pairs summing to `-nums[i]`. Three dedup points: outer skip on `nums[i] == nums[i-1]`, and after each hit advance `l`/`r` past clones of the used values.> — O(n²) time, O(1) extra (excl. sort stack and output)
- **Key insight:** <Sorting collapses two problems at once: (a) it enables the O(n) two-pointer inner scan (which only works on sorted input), and (b) it makes duplicates adjacent so dedup becomes a simple "skip if equal to neighbor" check. Without sort, dedup would need a hash set of sorted triples — same time class but uglier and slower. INNER POINTER DEDUP FIRES ONLY ON `SUM == 0` (FOUND TRIPLET)>
- **Edge cases I had to handle:** <(1) n < 3 → outer loop body never runs, returns []; (2) all zeros `[0,0,0,0]` → emits `[0,0,0]` once, outer dedup kills repeats; (3) all positives / all negatives → early exit on `nums[i] > 0` or two-pointer never converges; (4) heavy duplicates `[-2,-2,0,0,2,2]` → inner dedup loops prevent `[-2,0,2]` from being emitted twice; (5) anchor already > 0 → no possible triple, break early (sorted ⇒ all later values also positive).>
- **Where I got stuck and for how long:** <40+ mins, needed editorial. Had the O(n²) sort + two-pointer structure but couldn't reason through the dedup correctness. Root causes: (1) didn't realize there are **three** independent dedup points, not one; (2) confused "skip duplicate anchor" (check BEFORE work, `nums[i] == nums[i-1]`) with "skip duplicate after a hit" (check AFTER recording, `nums[l] == nums[l+1]`); (3) tried to dedup the result list post-hoc instead of preventing duplicates at generation time. Editorial showed: outer-dedup is process-then-skip-next; inner-dedup is record-then-skip-clones.>
- **Template fragments I reused:** <Sort + two-pointer kSum scaffold; the three-point dedup pattern (outer `if i > 0 && nums[i] == nums[i-1] continue`; inner `while l < r && nums[l] == nums[l+1] l++` after hit; mirror for r); early-exit on `nums[i] > 0`. Reusable for LC 16 (3Sum Closest), 18 (4Sum), 259 (3Sum Smaller), and kSum generalizations.>
