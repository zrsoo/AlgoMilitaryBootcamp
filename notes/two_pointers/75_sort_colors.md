# LC <75> — <Sort Colors>
- **Pattern:** <Two Pointers — Dutch National Flag three-way partition>
- **Brute force:** <Counting sort: pass 1 counts 0s/1s/2s, pass 2 overwrites the array in order> — O(n) time, O(1) space, **two passes**
- **Optimized:** <Dutch National Flag: three pointers `low`, `mid`, `high`. Scan with `mid`; swap 0s left with `low`, 2s right with `high`. Asymmetry: on a 2-swap `mid` stays put (newly swapped-in value is unknown).> — O(n) time, O(1) space, **one pass in-place**
- **Key insight:** <Maintain four regions by invariant: `[0..low-1]` are 0s (confirmed), `[low..mid-1]` are 1s (confirmed), `[mid..high]` is unknown, `[high+1..n-1]` are 2s (confirmed). Every iteration shrinks the unknown region by 1, guaranteeing O(n). The reason `mid` doesn't advance on a 2-swap: `nums[high]` was unknown before the swap, so the new `nums[mid]` is also unknown and must be re-inspected. The 0-swap is safe to advance because `nums[low]` was a confirmed 1 (or `low == mid`).>
- **Edge cases I had to handle:** <(1) n == 0 or 1 → loop condition `mid <= high` false immediately, no-op; (2) already sorted `[0,0,1,1,2,2]` → low/mid advance through 0s and 1s, high decrements through 2s without swaps doing anything destructive; (3) all same value → trivially one branch; (4) reverse-sorted `[2,2,1,1,0,0]` → forces maximal swap activity, still O(n); (5) `low == mid` early on → swap is a no-op on the same index, still correct.>
- **Where I got stuck and for how long:** <Could not solve without editorial. Knew counting sort works but the problem flagged "one-pass, constant space" as the real ask. Root cause: didn't recognize this as Dutch National Flag (Dijkstra). Once hinted at three-pointer partition, the asymmetry (`mid` advances on 0 but not on 2) was the second hurdle — had to derive it from the invariant that the swapped-in value from `high` is unknown territory.>
- **Template fragments I reused:** <Three-pointer partition scaffold (`low/mid/high`, `while mid <= high`); invariant-driven pointer advancement; in-place swap with temp. Reusable for: any 3-way partition (e.g., quicksort 3-way pivot for arrays with many duplicates), partitioning by predicate (negatives/zeros/positives), wiggle sort variants.>
